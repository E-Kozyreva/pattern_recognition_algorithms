kmeans = KMeans(n_clusters=10)
kmeans.fit(X)
pca = PCA(n_components=2)

plt.figure(figsize=(10, 8))
for i in range(10):
    plt.scatter(X_pca[kmeans.labels_ == i, 0], X_pca[kmeans.labels_ == i, 1], label=f'Cluster {i}', alpha=0.5)
plt.scatter(pca.transform(kmeans.cluster_centers_)[:, 0], pca.transform(kmeans.cluster_centers_)[:, 1], marker='x', color='red', label='Cluster centers')

plt.xlabel('Principal Component 1')
plt.ylabel('Principal Component 2')
plt.legend()

epsilon = [3, 4, 5]
res = np.empty((0, 2))
for epsi in epsilon:
    for min_semp in range(3, 50):
        dbscan = DBSCAN(eps=epsi, min_samples=min_semp)
        dbscan.fit(X)
        labels = dbscan.labels_
        n_clusters_ = len(set(labels)) - (1 if -1 in labels else 0)
        n_noise_ = list(labels).count(-1)
        if n_clusters_ == 10:
            res = np.vstack([res, [epsi, min_semp]])
        print("Оценка количества кластеров:", n_clusters_, "Оценка количества шумовых точек:", n_noise_)
        print("epsilon и min_samples:", epsi, min_semp)

Выводы¶

Плюсы DBSCAN:¶

  • Способность обнаруживать кластеры любой формы: DBSCAN может успешно выделять кластеры, которые имеют нелинейные формы или которые различаются по плотности.
  • Устойчивость к шуму: Благодаря своей основной идее о плотности точек, DBSCAN обычно не принимает шум за кластеры.
  • Не требует заранее определенного числа кластеров: В отличие от методов, таких как k-средних, DBSCAN не требует знания количества кластеров заранее.
  • Прост в реализации и понимании: Алгоритм относительно прост в реализации и понимании, поэтому его легко использовать для различных задач.

Минусы DBSCAN:¶

  • Чувствительность к параметрам: DBSCAN имеет два основных параметра: радиус эпсилон (epsilon) и минимальное количество точек (MinPts), и выбор правильных значений этих параметров может быть сложной задачей.
  • Неэффективность на данных высокой размерности: При работе с данными высокой размерности производительность DBSCAN может существенно снижаться из-за проклятия размерности.
  • Не подходит для разреженных данных: Если данные разрежены, DBSCAN может работать менее эффективно из-за того, что понятие плотности может стать менее информативным.
  • Проблемы с кластеризацией с различными плотностями: В случае, когда кластеры имеют сильно отличающиеся плотности, DBSCAN может плохо работать.
